First Person Movement in Unity

Overview

I remember always hearing people talking about how efficient C# is, and that its the best coding language. I had also seen it in action on the Unity editor a while back, when a friend of mine was messing around with VR. I wanted to be able to create something similar, so I started learning C# and Unity myself. I thought the most engaging way of doing this would be to create first person movement in the Unity editor because I've always enjoyed first person games, but never knew what was involved in making them.

Starting Out

I had zero knowledge of C#, outside of general coding knowledge I had accumulated over the years. I started with the basics, learning about variables, data types, and control structures. As I became more comfortable with the language, I began to explore Unity's features and how they could be used to create immersive experiences. I worked in three files: MoveCamera.cs, PlayerCam.cs, and PlayerMovement.cs. They defined how the camera was controlled, defined the actual camera properties, and defined how the player moved respectively.
